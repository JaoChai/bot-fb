# Dashboard Redesign — "Daily Ops, Clean Minimal"

**วันที่:** 2026-07-09
**สถานะ:** อนุมัติโดยเจ้าของแล้ว
**ขอบเขต:** หน้าแดชบอร์ด (frontend `DashboardPage` + backend `DashboardController` เฉพาะส่วน alerts) — ไม่แตะข้อมูลใน DB

## เป้าหมาย

เจ้าของเปิดแดชบอร์ดวันละครั้งเพื่อเห็นภาพรวมทุกด้านพร้อมกัน (ยอดขาย, ลูกค้ารอตอบ, สุขภาพบอท, ต้นทุน) — redesign ให้เรียงตามความเร่งด่วน ตัดส่วนที่ข้อมูลจริงใน DB บอกว่าไม่ได้ใช้ และแก้ตัวเลข handover ที่หลอกตา

## Layout ใหม่ (บน → ล่าง)

1. **Alert strip** — โชว์เฉพาะมีเรื่องจริง ไม่มีเรื่อง = ซ่อนทั้งแถบ
   - ลูกค้ารอมนุษย์ตอบ **ภายใน 48 ชม. เท่านั้น** — ชื่อลูกค้า + รอมานานเท่าไหร่ + กดไปหน้าแชท
   - บอทสถานะไม่ active → เตือนแดง
2. **Stat cards 4 ใบ** (ตัวเลขใหญ่ อ่านชัด)
   - ยอดขายวันนี้ (+จำนวนออเดอร์, trend เทียบเมื่อวาน)
   - ยอดขายเดือนนี้ (+ชิป VIP: 198 คน / ยอดซื้อรวม)
   - ข้อความวันนี้ (trend เทียบเมื่อวาน)
   - ค่า API วันนี้ (บรรทัดรอง: ยอดเดือน + เฉลี่ยต่อข้อความ — ยุบ CompactCostBreakdown เข้ามา)
3. **กราฟยอดขาย vs ค่า API** — dual axis, ตัวเลือกช่วง 7/14/30 วัน, เอา VIP ออกจากกราฟ
4. **สินค้าขายดี + ออเดอร์ล่าสุด** — สองคอลัมน์ (จอเล็กเรียงลง)
5. **ท้ายหน้า:** สถานะบอทแบบย่อ + สวิตช์สต็อก (เฉพาะ owner)

## ตัดออก

- `RecentActivityTimeline` (ข้อมูลบางเกิน)
- `CompactCostBreakdown` (ซ้ำกับการ์ดค่า API — ยุบรวม)
- `BusinessHealthBar` เดิม (แทนด้วย alert strip ที่กรองแล้ว)
- VIP บนกราฟ dual-axis

## Backend เปลี่ยนแปลง

- `DashboardController::getAlerts()` — เพิ่มเงื่อนไข `updated_at >= NOW() - 48 hours` กับ handover conversations
- `summary.handover_conversations` — นับด้วยเงื่อนไขเดียวกัน (ตัวเลขบน alert strip ตรงกับรายการ)
- **ไม่แตะข้อมูลใน DB** — conversation เก่ายังสถานะ handover เหมือนเดิม เห็นได้ในหน้าแชท

## สไตล์

- Clean minimal ต่อยอด shadcn/Tailwind ธีมเดิม — ขาว/ดำ, ตัวเลขใหญ่, ช่องไฟเยอะ, ไอคอนเส้นบาง
- สีเฉพาะจุดมีความหมาย: เขียว = ทำงาน/กำไร, แดง = ต้องดูแล
- รองรับ dark mode + มือถือ (การ์ด 2 คอลัมน์บนจอเล็ก)

## การทดสอบ

- PHPUnit: `DashboardApiTest` — handover เก่ากว่า 48 ชม. ไม่โผล่ใน alerts/summary, ภายใน 48 ชม. โผล่
- Vitest: component ใหม่ render ถูกต้อง, alert strip ซ่อนเมื่อไม่มี alert
- ตรวจภาพจริงบน browser: light/dark + มือถือ

## Error handling

- แต่ละส่วนโหลดแยก (React Query เดิม) — ส่วนไหน error ไม่ล้มทั้งหน้า, คงหน้า error รวม + ปุ่มลองใหม่เดิมไว้เฉพาะกรณี summary หลักล้ม
