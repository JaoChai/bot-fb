# MCP Configuration

This project uses MCP (Model Context Protocol) servers for AI agent tools (Claude Code, pi, Codex, etc.).

## Setup

1. **Copy the example to a real config:**
   ```bash
   cp .mcp.example.json .mcp.json
   ```

2. **Replace placeholders** in `.mcp.json` with real keys from each provider:

   | Placeholder | Where to get the key |
   |-------------|---------------------|
   | `<NEON_API_KEY>` | https://console.neon.tech/app/settings/api-keys |
   | `<SENTRY_ACCESS_TOKEN>` | https://sentry.io/settings/account/api/auth-tokens/ |
   | `<OPENAI_API_KEY>` | https://platform.openai.com/api-keys |
   | `<CONTEXT7_API_KEY>` | https://context7.com/dashboard |

3. **`.mcp.json` is gitignored** — keys stay local. Never commit it.

## Why two files?

- `.mcp.example.json` → committed, shows the config structure
- `.mcp.json` → local only (gitignored), contains real keys

## Security note

Historical commits in this repo contained real keys before they were untracked.
If you cloned an old version, those keys may be visible in `git log`. They
have been rotated — only the keys in your current local `.mcp.json` are valid.
